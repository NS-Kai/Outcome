1. Explain the importance of codes readability.
    • Code with good readability is better for us, as this may bring some benefit for us. First, it allow us have a better understand about how the code is working. Then, it can also make us easily in debug process. If these code can't work properly, a good readability of code can make us easy to understand and find out what is causing it fault. Moreover, it also make us easy to maintain it, and easy to extend the code.
  2. Identify the modern coding guides for web pages development.
    • Modern coding guides can let most of the programmer can read and understand the code easily. When in web pages development, have a good code readability will be easily for all programmer. However, if we just use and follow own coding guides, it may confuse other programmer, and make the web pages development process delay.
  3. Introduce one of your favourites coding guide standard.
    • Coding guide standard is likely coding conventions, which is the habit when coding. My favourites coding guide standard is using tab button, which has same effect with 4 times of space. This will make the whole coding become tidy and can have good code readability.
  4. Introduce the tools used to debug CSS/HTML problems.

    • Git is one of the tools that can used to debug CSS and HTML problems. It provided a platform for a team, which allow them can debug CSS and HTML problems together. In the past, these kind of project or work also work as a team, but the difference is they may need to do the work together and discussion together, so they can get a unified option. With Git, anyone can do the debug at anywhere, anytime. Plus, the debug result will be show to every teammates, and the debug part will be shown also.
  5. Explain how version control tools can solve team collaboration problems

    • Version control has record the changes of a file or set of files that we can recall the specific version later. When a team need to develop a program, version control provide good help, as any changes about the program will be shown to other teammates, so they just need to refer on it and won't repeat the same work again.
